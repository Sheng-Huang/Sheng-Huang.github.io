第三章 对象

数据存储

  • 数据存储的位置:
    1. Registers:位于CPU中，java对寄存器没有控制权
    2. Stack:位于RAM中，通过栈指针获得处理器的直接支持。java必须知道栈内保存的所有项的生命周期，因此对象本身不适合保存在栈上
    3. heap：位于RAM中，存储所有java对象，不需要知道对象的生命周期。当new一个对象时，自动在堆中进行内存分配。另外，由于堆是线程共享的，方便于多线程访问对象
    4. constant storage :通常存放在程序代码中。如需严格保护，可将他们置于ROM中
    5. Non-RAM storage： 数据完全存在于程序之外，在程序未运行以及脱离程序控制后依然存在。例如序列化对象被转化为字节流，发送给另一台机器。例如持久化对象被放置在磁盘上。这种存储方式是将对象转存于另一介质中，并在需要时恢复成常规的，基于RAM的对象

基本类型的存储

  • java采用了和C++相同的策略，即使用一个自动变量直接存储值，并置于栈内存中
  • java确定了每种基本类型的内存占用大小。这些大小不会像其他一些语言那样随着机器环境的变化而变化 如果需要在堆内存中表示基本数据类型，需要用到他们的包装类:

自动装箱：

自动拆箱：

1

char c = ch;
  • 为了保证安全性，类中的基本类型会在未显示初始化时为默认值。方法不属于类，因此不会默认初始化

高精度数值

  • 在 Java 中有两种类型的数据可用于高精度的计算。它们是BigInteger和BigDecimal

数组的存储

  • 基于安全性，java中使用数组必须初始化，并且不能访问长度以外的数据
  • 当创建一个数组时，实际上是创建了一个引用数组，并且每个引用的初始值为null。由上一点知，如果我们尝试使用为 null 的引用，则会在运行时报错
  • 我们还可创建基本类型的数组。编译器通过将该数组的内存全部置零来保证初始化

  • 作用域定义了该范围内定义的变量名的可见性和生存周期。一般由大括号决定
  • 有别于C和C++可隐藏较大作用域变量，java不能在作用域内重定义变量，如下:

  • java对象与基本类型具有不同的生命周期。当我们使用new关键字来创建java对象时，它的生命周期将会超出作用域，而它的引用则在作用域终点结束。java的垃圾收集器会检查所有new出来的对象并判断哪些不再可达，继而释放那些被占用的内存，供其他新的对象使用
  • return结束方法同时返回值

程序编写

命名可见性

C++ 将函数嵌套在类中，所以它们不会和嵌套在其他类中的函数名冲突。然而，C++还是允许全局数据和全局函数，因此仍有可能发生冲突。为了解决这个问题，C++ 使用附加的关键字namespace引入了命名空间

java 采取了一种新的方法避免了以上这些问题：为一个类库生成一个明确的名称，java 创建者希望我们反向使用自己的网络域名，因为域名通常是唯一的

使用反向 URL 将命名空间与文件路径相关联不会导致BUG，但它却给源代码管理带来麻烦。例如在 com.mindviewinc.utility.foibles 这样的目录结构中，我们创建了com和mindviewinc空目录。它们存在的唯一目的就是用来表示这个反向的 URL

使用其他组件

无论何时在程序中使用预先定义好的类，编译器都必须找到该类。最简单的情况下，该类存在于被调用的源代码文件中。此时我们使用该类 —— 即使该类在文件的后面才会被定义（Java 消除了所谓的“前向引用”问题）

当类在其他地方时，必须通过使用import关键字来告诉 java 编译器具体要使用的类。import指示编译器导入一个包，也就是一个类库（在其他语言中，一个库不仅包含类，还可能包括函数和数据，但请记住 Java 中的所有代码都必须写在类里）。大多数时候，我们都在使用 Java 标准库中的组件。有了这些构件，就不必写一长串的反转域名

static

  • static创建不需要类和对象的字段或方法，static字段和方法可以用类名+.+字段/方法表示
  • static也可创建static对象与嵌套类

第四章 运算符

一些运算符

  • Side Effect指的是运算符可以改变运算对象的值
  • 基本类型的赋值都是直接的，而不像对象，赋予的只是其内存的引用，因此传引用后改变所指向的同一个对象的值。
  • equals()的默认行为是比较对象的引用而非具体内容。除非你在新类中覆写 equals() 方法，否则我们将获取不到想要的结果
  • 逻辑运算符支持一种称为短路（short-circuiting）的现象。整个表达式会在运算到可以明确结果时就停止并返回结果，这意味着该逻辑表达式的后半部分不会被执行到。
  • 在java中，不能像C/C++一样对非布尔值使用逻辑运算符。
  • Java 也添加了一种“不分正负”的右移位运算符（»>），它使用了“零扩展”（zero extension）：无论正负，都在高位插入 0。这一运算符是 C/C++ 没有的。
  • 如果移动 char、byte 或 short，则会在移动发生之前将其提升为 int，结果为 int。仅使用右值（rvalue）的 5 个低阶位，即其能表示的int的最大位数。这可以防止我们移动超过 int 范围的位数。若对一个 long 值进行处理，最后得到的结果也是 long。
  • 三元运算符：布尔表达式 ? 值 1 : 值 2 若表达式计算为 true，则返回结果 值 1 ；如果表达式的计算为 false，则返回结果 值 2。

转换相关

  • 类似移位，char、byte 或 short，会在计算发生之前将其提升为 int。
  • java中，narrowing conversion时编译器强制显式转换。
  • 使用整型数值类型时，显示其二进制形式会很有用。在 Long 型和 Integer 型中这很容易实现，调用其静态的toBinaryString()方法即可。 但是请注意，若将较小的类型传递给 Integer.toBinaryString() 时，类型将自动转换为 int。
  • 如果在预期为 String 类型的位置使用 boolean 类型的值，则结果会自动转为适当的文本格式（即 “true” 或 “false” 字符串）。同时，当表达式不为布尔值时，java不会强制转换其为布尔值。

基本类型相关

  • Java 7 引入了二进制的字面值常量，由前导 0b 或 0B 表示，它可以初始化所有的整数类型。
  • 整数除法会直接砍掉小数，而不是进位。
  • 从 float 和 double 转换为整数值时，小数位将被截断。若你想对结果进行四舍五入，可以使用 java.lang.Math 的round()方法。
